## Branch structure:

- master: main branch, you can push all of the changes which is ready/finished
- preview: preview branch, this branch is good in cases like: I have project in the internet (production) and i want check how something will looks in the internet
- develop: this branch is using to working on creating some section, things etc. just work

E.g.: You are working on develop branch and finished some section, so you can push changes to master (but i prefer to push to *preview*) branch

## Files structures:

1. main:

   - backend – for Payload CMS (CMS)
   - frontend-new – for Next.js (frontend application)

2. frontend:
   - src/
     - app/
       - assets/: project assets (e.g. images, for section on mainpage and subpages)
         - subpages/: assets for subpages (e.g. error, 404)
       - components/: main page components (e.g. components for mainpage)
       - UI/: UI components used multiple times (e.g. buttons, dialogs, mobile navigation)
       - styles/: global styles and SCSS/CSS variables
         - \_colors.scss: color variables
         - \_components.scss: reusable classes
         - \_functions.scss: SCSS functions
         - \_variables.scss: SCSS variables
         - globals.css: global styles (reset styles)
       - utils/: tools and helper functions for the app layer
       - icons/: SVG/TSX icons
       - constants/: project constants (e.g. accordionData.ts or navigationData.ts)
       - api/: API request handling
       - (subpages)/: project subpages
         - podstrona/: example subpage
           - page.tsx: subpage file
             Structure of subpages:
             subpage-name (folder):
             - page.tsx
             - subpage-name.module.scss: I am using the same name like folder name but in english, not page.module.scss
             - /components (folder, if required)
       - page.tsx: main page
       - layout.tsx: main layout
       - not-found.tsx, error.tsx: error handling
       - favicon.ico, 404.module.scss: favicon and 404 styles
     - componentsShadc/: UI components generated by shadcn (e.g. button, dialog, navigation-menu, etc.)
     - utils/: general utilities (e.g. fetch-api, get-strapi-url, getImageWithBlur, types)
     - lib/: helper functions (e.g. stripe, utils, currency conversions)
     - data/: data files (e.g. loaders.ts)
